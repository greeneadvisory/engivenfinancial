Request Body
Note: All queries are case-insensitive
Key Pairs
A key pair can be used to search records for values matching exactly what is specified. 


{
    “donorName”: “Faith Kim”
}



You can search for multiple attributes. All attributes must match exactly (unless also using the modifiers described below).


{
    “donorName”: “Faith Kim”,
    “npoName”: “My Church”
}



In Pairs
Provide an array to find records whose value for this attribute exactly matches any of the specified search terms




{
    “donorName”: [ “Faith” , “Nathan” ]
}



Or Pairs
Use the “or” modifier to match any of the nested rulesets you specify as an array of query pairs. For records to match an or query, they must match at least one of the specified query pairs in the or array.


{
    “or” : [
        { “donorName”: “Faith” },
        { “donorName”: “Nathan” },
        { “npoName”: “Test Org” },
    ]
}

( The above means search for records that have “Faith” as donorName, OR have “Nathan” as donorName, OR has “Test Org” as the npoName )


Modified Pairs
Modified pairs are useful when a strict equality check wouldn't work.


{
    “donorName”: { “contains”: “Faith” }
}



List Of Modifiers
* “<” , “>” , “<=” , “>=”  will only work for dates. Currently in our database numeric amounts are stored as strings, so comparisons will not work as expected
* Dates are stored in UTC time, so you will need to make the correct adjustments to compare by Pacific Time. 
Refer to “<” for examples


“<”   /   “lessThan”


{
    “transactionConfirmedTimeStamp”: { “<”: “2026-01-01” }
}





Or


{
    “transactionConfirmedTimeStamp”: {
        “<”: “2025-01-01T08:00:00.000Z”
    }
}



Or, if using Javascript 


{
    “transactionConfirmedTimeStamp”: {
        “<”: new Date('2026-01-01T00:00:00-08:00')
    }
}

“<=”  /  “lessThanOrEqual”


{
    “transactionConfirmedTimeStamp”: { “<=”: “2026-01-01” }
}

“>”  /  “greaterThan”


{
    “transactionConfirmedTimeStamp”: { “>”: “2026-01-01” }
}

“>=”  /  “greaterThanOrEqual”


{
    “transactionConfirmedTimeStamp”: { “>=”: “2026-01-01” }
}

** Ranges
You can do range queries using the comparison operators


{
    “transactionConfirmedTimeStamp”: { 
        “>=”:  “2025-01-01T08:00:00.000Z”,
        “<”: “2026-01-01T08:00:00.000Z”
    }
}

“!”  /  “not”
Searches for records that are not equal to the value specified




{
    “npoName”: { “!”: “Test Org” }
}



You can also provide an array to find records that are not exact matches to any of the specified terms


{
   “donorName”: { “!”: [ “Faith” , “Nathan” ] }
}

“like”
Searches for records using pattern matching using “%” to represent wildcard. For example:
“%test” → ends with “test”
“test%” → starts with “test”
“%test%” → contains “test”
“%test%match” → contains “test” and ends with “match”




{
    “npoName”: { “like”: “%Test” }
}

“contains”
Searches for records that contains the given string


{
    “npoName”: { “contains”: “Test” }
}

“startsWith”
Searches for records that starts with the given string


{
    “npoName”: { “startsWith”: “Test” }
}

“endsWith”
Searches for records that ends with the given string


{
    “npoName”: { “endsWith”: “Test” }
}

